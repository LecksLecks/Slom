# Стратегия «Слом тренда v11»

Стратегия-обёртка над индикатором [`slom_trenda_v11.pine`](../../slom_trenda_v11.pine).
Файл: [`slom_trenda_v11_strategy.pine`](../../slom_trenda_v11_strategy.pine).

> ⚠️ Это инженерия стратегии и обучение, **не финансовая рекомендация**.
> Бэктест **не является** обещанием live-результата. Цифры производительности
> здесь **не приводятся**: Pine исполняется только на TradingView, локально его
> прогнать нельзя — ниже дан протокол, снимите метрики сами.

## Гипотеза эджа

После подтверждённого **слома структуры** (пробой значимого экстремума с
«закрепом» полнотелыми свечами) цена продолжает движение в сторону слома
достаточно часто, чтобы при фиксированном R:R и структурном стопе давать
положительное матожидание. Полный скор (все включённые фильтры пройдены)
отсеивает слабые/ложные сломы.

## Как устроено (переиспользование, не переписывание)

Стратегия — это **точная копия индикатора** с двумя изменениями, поэтому её
сигналы бит-в-бит совпадают с индикатором:

1. Объявление `indicator(...)` заменено на `strategy(...)` с реалистичными
   параметрами исполнения.
2. В конец добавлен торговый блок (группа настроек «8. Стратегия»), который
   читает уже посчитанные сигналы `emitLocUp/Dn`, `emitGlobUp/Dn` и структурные
   уровни `sFarLocUp/Dn`, `sFarGlobUp/Dn`.

Вся визуализация индикатора (зоны, метки, панель тренда) сохранена.

## Правила входа/выхода

- **Лонг:** на закрытии подтверждённого бара с сигналом слома ВВЕРХ
  (`emitGlobUp` или `emitLocUp`), если нет встречного сигнала.
- **Шорт:** симметрично по слому ВНИЗ.
- **Стоп (структурный):**
  - лонг: `sFar…Up − slBufferViz × ATR14`
  - шорт: `sFar…Dn + slBufferViz × ATR14`
  - глобальный уровень приоритетнее локального.
- **Тейк:** от входа на расстоянии `stopDist × rrRatioViz` (R:R по умолчанию 3.0).
- Вход только если `stopDist > 0` (стоп по правильную сторону от цены).
- Реализовано через `strategy.entry` + `strategy.exit` (OCO: stop + limit).

## Параметры (ключевые)

| Параметр | По умолчанию | Смысл |
|---|---|---|
| `stratEnabled` | true | Включить исполнение |
| `stratSide` | Обе | Обе / Только лонг / Только шорт |
| `useLocSig` / `useGlobSig` | true / true | Торговать локальные / глобальные сломы |
| `allowReverse` | true | Разворот по встречному сигналу vs вход только из флэта |
| `gateSignalsByScore` | true | **Держать ВКЛ** — торговать только полный скор |
| `rrRatioViz` | 3.0 | R:R тейка |
| `slBufferViz` | 0.3 | Буфер стопа за зоной, × ATR14 |
| `useDates` | false | Ограничить окно бэктеста |

Настройки исполнения `strategy()`: капитал 10 000, размер позиции 10% от эквити,
комиссия 0.05% (taker-подобно для крипты), проскальзывание 2 тика,
`pyramiding = 0`, `process_orders_on_close = true`.

## Статус перерисовки (repaint) / lookahead

**Не перерисовывает.** Обоснование:

- Все сигналы формируются внутри `if barstate.isconfirmed` — т.е. на **закрытии**
  бара, а не внутри его.
- Входы исполняются на закрытии подтверждённого бара
  (`process_orders_on_close = true`, `calc_on_every_tick = false`).
- Единственные `request.security` (HTF-EMA и OI) берут значения со сдвигом `[1]`
  и с `barmerge.lookahead_off` → без заглядывания в будущее.

Проверьте на реальном инструменте, что метки/сделки на закрытых барах не смещаются.

## Протокол бэктеста (снять на TradingView)

1. Добавить `slom_trenda_v11_strategy.pine` на график, вкладка **Strategy Tester**.
2. **Инструмент/ТФ:** индикатор рассчитан на структуру **1H** — тестировать на 1H
   (напр. `BINANCE:BTCUSDT`, `BINANCE:ETHUSDT`).
3. **Период:** максимально доступная история 1H; отдельно прогнать **out-of-sample**
   (например, обучение 2023–2024, проверка 2025–2026) через `useDates`.
4. **Настройки:** `gateSignalsByScore = ВКЛ`; комиссия/слиппедж уже в коде — при
   необходимости привести к своей бирже.
5. **Снять метрики:** Net Profit, Max Drawdown, Profit Factor, Win Rate,
   Total Closed Trades, Avg Trade, Sharpe/Sortino.
6. **Чувствительность:** прогнать `rrRatioViz ∈ {2, 3, 4}`, `slBufferViz ∈ {0.2, 0.3, 0.5}`,
   сравнить только лонг / только шорт / обе стороны.

Значимость требует **достаточной выборки** (ориентир — от ~100 закрытых сделок).

## Честные ограничения

- **Оверфит:** фильтры и R:R легко подогнать под один период — обязателен
  out-of-sample / walk-forward, иначе метрики недостоверны.
- **Режимы рынка:** пробойная логика деградирует во флэте/пиле (серии ложных
  сломов и стопов). Смотрите просадку именно в боковиках.
- **Микроструктура:** комиссия/проскальзывание заданы усреднённо; на неликвиде и
  на младших ТФ реальные издержки выше.
- **Исполнение на закрытии бара:** консервативно и без заглядывания, но реальный
  вход по рынку может отличаться от цены закрытия сильнее заложенного слиппеджа.
- **OI-фильтр** требует наличия символа `<тикер>_OI` у брокера, иначе не влияет.
