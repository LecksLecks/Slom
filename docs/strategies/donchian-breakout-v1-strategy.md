# Стратегия «Donchian Breakout v1»

Самостоятельная трендовая стратегия (не адаптация индикатора).
Файл: [`donchian_breakout_v1.pine`](../../donchian_breakout_v1.pine).

> ⚠️ Это инженерия стратегии и обучение, **не финансовая рекомендация**.
> Бэктест **не является** обещанием live-результата. Цифры производительности
> здесь **не приводятся**: Pine исполняется только на TradingView — прогоните
> протокол ниже и снимите метрики сами.

## Гипотеза эджа

Пробой N-барового экстремума маркирует начало направленного импульса. У BTC
такие импульсы дают достаточно длинный «хвост» прибыли, чтобы один-два больших
тренда окупили серию мелких ложных пробоев. Эдж — не в проценте выигрышей
(он низкий), а в **асимметрии**: маленькие частые убытки против редких крупных
прибылей, которым дают течь.

## Правила входа/выхода

| | Условие |
|---|---|
| **Лонг** | `close > highest(high, N)[1]` (закрытие выше хая N прошлых баров), позиция плоская |
| **Шорт** | `close < lowest(low, N)[1]`, позиция плоская |
| **Выход** | **Единственный** — chandelier ATR-трейлинг-стоп |
| Long-стоп | `max(prev_stop, highestSinceEntry − atrMult × ATR)` — двигается только вверх |
| Short-стоп | `min(prev_stop, lowestSinceEntry + atrMult × ATR)` — только вниз |

- Вход исполняется на **закрытии** пробойного бара (`process_orders_on_close`).
- **Фиксированного тейка нет намеренно** — он обрезал бы правый хвост тренда,
  который и оплачивает всю систему. Позицию закрывает только трейлинг-стоп.
- Вход **только из флэта**: без пирамидинга и без разворота на встречном
  сигнале — одно решение меньше = один параметр меньше.

## Параметры и почему их так мало

| # | Параметр | Дефолт | Роль |
|---|---|---|---|
| 1 | `dcLen` — длина канала N | 20 | Основной тюнинг (классика turtle) |
| 2 | `atrMult` — множитель ATR-трейла | 3.0 | Основной тюнинг (ширина стопа) |
| 3 | `atrLen` — период ATR | 14 | Стандарт; обычно НЕ трогать |
| 4 | `tradeDir` — Обе / Лонг / Шорт | Обе | Режим (структурный, не число) |

Отдельная группа «Бэктест-окно» (`useDates/startDate/endDate`) — **не эдж-параметры**,
только для OOS/walk-forward прогонов; на сигналы в live не влияют.

**Почему минимум.** Реально настраиваются лишь 2 числа (`dcLen`, `atrMult`).
По правилу «число параметров ≪ √(число сделок)»: при ~100+ сделках порог ≈ 10,
у нас 2 → низкий риск переобучения. Любой фильтр «для качества сигнала» (EMA,
объём, HTF, сессии, ретест) **сознательно не добавлен** — это противоречит цели
устойчивости и раздувает пространство подгонки.

## Направление: обоснование

Дефолт — **обе стороны**, но направление вынесено в инпут:
- У BTC исторический up-drift → на длинной истории **long-only** обычно даёт
  лучший risk-adjusted профиль; short-breakout страдает от резких шорт-сквизов.
- Но медвежьи тренды (напр. 2022) ловятся только шортами.
- Параметр **структурный, не подгоночный** — пусть выбор подтвердит бэктест на
  вашем окне (сравните три режима, см. протокол).

## Настройки исполнения `strategy()`

Капитал 10 000, размер позиции 10% от эквити, комиссия 0.05% (taker-подобно для
крипты), проскальзывание 2 тика, `pyramiding = 0`,
`process_orders_on_close = true`, `calc_on_every_tick = false`.
Сайзинг фиксированным % — для демонстрации; в live разумнее риск-сайзинг по ATR.

## Статус перерисовки (repaint) / lookahead

**Не перерисовывает.** Обоснование:

- Уровни канала берутся со сдвигом `[1]` (`highest(high,N)[1]`, `lowest(low,N)[1]`)
  → рассчитаны только по **закрытым** барам, текущий бар исключён.
- Вход по `close` подтверждённого бара при `process_orders_on_close = true` и
  `calc_on_every_tick = false` — без внутрибарового заглядывания.
- **Нет `request.security`** → нет источника lookahead с других ТФ.
- Трейлинг-стоп на бар *t* рассчитан по данным ≤ `close(t)` и активен на барах
  *> t*; исполняется **intrabar по касанию** (реалистичное поведение стоп-ордера).

Проверьте на живом инструменте, что метки/сделки на закрытых барах не смещаются.

## Протокол бэктеста (снять на TradingView)

1. Добавить `donchian_breakout_v1.pine` на график, вкладка **Strategy Tester**.
2. **Инструмент/ТФ:** `BINANCE:BTCUSDT.P` (или спот `BINANCE:BTCUSDT`), **1H**.
3. **Период:** максимально доступная история 1H. Обязательно разбить на
   **in-sample / out-of-sample** через `useDates` (напр. подбор параметров на
   2023–2024, проверка на 2025–2026). Метрики OOS — решающие.
4. **Снять метрики:** Net Profit, Max Drawdown, Profit Factor, Win Rate,
   Total Closed Trades, Avg Trade, Sharpe, Sortino.
5. **Чувствительность (walk-forward мышление):** прогнать
   `dcLen ∈ {10, 20, 40, 55}`, `atrMult ∈ {2, 3, 4}` — метрики должны меняться
   **плавно**, а не иметь одинокий пик (пик = подгонка).
6. **Направление:** сравнить Обе / Только лонг / Только шорт на одном окне.
7. **Значимость:** ориентир — **от ~100 закрытых сделок**, иначе метрики шумовые.

Дальше отдать `quant-analyst` на OOS / walk-forward / Monte-Carlo и вердикт по overfitting.

## Честные ограничения

- **Флэт/пила — враг №1.** Breakout деградирует в боковике: серии ложных пробоев
  и выбитых стопов. Смотрите просадку именно на боковых участках (напр. лето 2023).
- **Низкий Win Rate — это норма** для трендследования (часто 30–45%). Система
  живёт за счёт редких крупных трендов; несколько пропущенных больших движений
  ломают весь результат за период.
- **Оверфит:** даже 2 параметра можно подогнать под одно окно — без OOS цифрам
  верить нельзя.
- **Проскальзывание:** на пробое ликвидность часто хуже средней; заложенные
  2 тика могут недооценивать реальные издержки на импульсе.
- **Исполнение на закрытии бара** консервативно, но реальный рыночный вход может
  отличаться от цены закрытия сильнее заложенного слиппеджа.
- **Бэктест ≠ live.** Funding у перпетуала не моделируется; на длинных
  удержаниях он влияет на итог.
